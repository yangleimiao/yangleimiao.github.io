spring aop简单使用

发表于 2022-07-17

AOP中的概念

连接点

类中可以被增强的方法称为连接点；

切入点

实际被增强的方法，称为切入点；

通知（增强）

实际增强的逻辑部分，也称为通知；
通知类型有多种，指增强的逻辑部分执行的位置：

  • 前置通知：方法执行之前先执行增强的逻辑；
  • 后置通知：方法执行之后再执行增强的逻辑（如果执行中发生异常，这里就不会执行）；
  • 环绕通知：方法执行之前和之后都执行增强的逻辑；
  • 异常通知：方法执行发生异常才执行增强的逻辑；
  • 最终通知：相当于try catch finally中的的finally，有没有异常都会执行；

切面：可以当做把通知应用到切入点的过程；

Spring AOP操作

Spring中一般基于AspectJ实现AOP操作

切入点表达式
语法：execution([权限修饰符][返回类型] [类全路径] 方法名称)
举例：execution(* com.demo.dao.User.add(..))

返回类型可以省略，参数类型可以用 .. 表示，类全路径、方法名称用 . 隔开；

示例

依赖

import org.springframework.stereotype.Component;
@Component
public class User {

public void add(int a,int b){
    System.out.println("add--"+a+b);
}

}

import org.aspectj.lang.annotation.Before;
import org.springframework.stereotype.Component;
import org.aspectj.lang.annotation.Aspect;
@Component
@Aspect
public class UserProxy {
//ProceedingJoinPoint is only supported for around advice
// 前置通知
@Before(value = “execution(* org.chengdu.aopanno.User.add(..))”)
public void before(){
System.out.println(“before method –”);
}
}

// 前置通知
@Before(value = "pointcut()")
public void before(){
    System.out.println("before method --");
}
// 后置通知
@AfterReturning(value = "pointcut()")
public void afterReturning(){
    System.out.println("after method return --");
}

// 最终通知 相当于finally
@After(value = "pointcut()")
public void after(){
    System.out.println("after method --");
}
// 异常通知
@AfterThrowing(value = "pointcut()")
public void afterThrowing(){
    System.out.println("afterThrowing --");
}

@Around(value = "pointcut()")
public void around(ProceedingJoinPoint joinPoint) throws Throwable {
    System.out.println("around method ["+joinPoint.getSignature().getName()+"] --before");
    joinPoint.proceed();
    System.out.println("around method ["+joinPoint.getSignature().getName()+"] --end");
}

}
